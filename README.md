A LangChain-based AI Agent that answers queries about Bangladesh using three local SQLite databases and live web search.

---

## 🎯 Project Overview

This agent uses **LangChain AgentExecutor** with **Groq (Llama 3.3 70B)** as the LLM brain. It intelligently routes queries to the correct tool:

| Tool | Purpose | Dataset Size |
|------|---------|-------------|
| `InstitutionsDBTool` | Schools, colleges, govt institutions | 34,901 records |
| `HospitalsDBTool` | Hospitals, clinics, health centers | 38,886 records |
| `RestaurantsDBTool` | Restaurants and eateries | 12,703 records |
| `WebSearchTool` | General knowledge, policies, current events | Live internet |

## ⚙️ Setup Instructions

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/bangladesh_agent.git
cd bangladesh_agent
```

### 2. Create and activate virtual environment
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Mac/Linux
python -m venv venv
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up API keys
Create a `.env` file in the root directory:
```
GROQ_API_KEY=your_groq_api_key_here
TAVILY_API_KEY=your_tavily_api_key_here
```

Get your free API keys:
- **Groq** (free LLM): https://console.groq.com
- **Tavily** (free web search): https://app.tavily.com

### 5. Build the databases (run once)
```bash
python setup_databases.py
```
This downloads the 3 HuggingFace datasets and creates the SQLite `.db` files in the `data/` folder.

### 6. Run the agent
```bash
python main.py
```

## 🧠 How It Works

```
User Question
      ↓
LangChain AgentExecutor
      ↓
LLM (Llama 3.3 70B) reads tool descriptions
      ↓
Picks the right tool + writes SQL or search query
      ↓
Tool executes → returns raw results
      ↓
LLM formats into natural language answer
      ↓
User sees clean answer
```

**The reasoning loop:**
1. User asks a question
2. AgentExecutor sends it to the LLM
3. LLM reads all 4 tool docstrings and decides which to use
4. For DB tools → LLM writes a SQL query → tool runs it on SQLite → returns rows
5. For WebSearchTool → LLM writes a search query → Tavily searches the internet → returns summaries
6. LLM formats the raw result into a clean natural language answer
